# home-stack

Single-node self-hosted stack on an OVH bare-metal VPS. Traefik is the single entry point for all services — TLS, JWT validation, and OIDC browser auth are all handled at the gateway. Individual services trust forwarded identity headers and never validate tokens themselves.

## Architecture

```
Internet (HTTPS :443)
        │
        ▼
   [Traefik]  — TLS (Let's Encrypt), JWT middleware (M2M), ForwardAuth (browser)
        │
        ├──▶ tasks-api      (tasks.DOMAIN/api)   — REST + MCP, JWT-protected
        ├──▶ keycloak        (auth.DOMAIN)         — OIDC provider, DCR endpoint
        ├──▶ oauth2-proxy    (private.DOMAIN)      — browser OIDC login flow
        └──▶ future services

[PostgreSQL]  — primary datastore for all services including Keycloak
[Redis]       — oauth2-proxy session store
[Watchtower]  — polls GHCR for first-party image updates only
```

## Auth flows

**Browser routes** (SPA → API): Traefik ForwardAuth → oauth2-proxy → Keycloak login → JWT issued → Go API validates JWT locally via cached JWKS.

**M2M routes** (MCP clients): Client registers via Keycloak DCR → gets `client_id`/`client_secret` → `client_credentials` grant → Bearer token → Traefik JWT middleware validates → Go API validates claims locally.

## Toolchain

| Tool | Role |
|---|---|
| Ansible | Host-level only: Docker install, systemd service, repo clone. Never calls service APIs. |
| OpenTofu | Service-level: Keycloak realm/clients/DCR, PostgreSQL roles/DBs, writes `/etc/home-stack.env`. |
| SOPS + age | Secrets at rest. Single keypair, no daemon. |
| Taskfile | Single entrypoint for all operations. |

## Key invariants

- Ansible scope: host-level only. Never calls service APIs.
- OpenTofu scope: service-level only. Never installs system packages.
- Secrets in repo: SOPS-encrypted (prod) or committed plaintext defaults (local) only. No hardcoded secrets anywhere else.
- Terraform state: never committed. Backblaze B2 only.
- Third-party images: pinned to explicit version tags, excluded from Watchtower.
- JWT validation: local in Go APIs via cached JWKS. No per-request Keycloak calls.
- CI tools: plain CLI calls, no GitHub-specific Action wrappers.

## Secrets model

**Prod**: `secrets/prod.sops.yaml` — SOPS-encrypted, committed. Contains operator-supplied inputs. Age private key never enters the repo.

**Runtime**: Generated by OpenTofu (`random_password`), written to `/etc/home-stack.env` on the VPS by the `stack` Ansible role. Never touch git or Ansible vars.

**Local**: `secrets/local.defaults.yaml` (committed plaintext defaults) + `secrets/local.override.yaml` (gitignored, sparse overrides). The Taskfile merges them before passing to OpenTofu.

## Commands

```bash
task install            # install uv-managed dev tooling and lefthook pre-commit hook
task dev                # start local stack, provision, follow logs
task dev:start          # start local stack and run provisioning
task dev:logs           # follow local stack logs
task dev:stop           # stop local containers without deleting volumes
task local:reset        # tear down volumes and generated state

task check              # format + lint + validate (pre-commit)
task format             # yamllint + tofu fmt
task lint               # yamllint + ansible-lint
task validate           # ansible syntax + tofu validate + compose config

task bootstrap          # one-time VPS bootstrap (manual only, see ansible/bootstrap.yml)
task provision          # decrypt prod secrets + tofu apply (requires SSH tunnel on :15432)
task deploy             # ansible site.yml against prod
task rotate             # provision + deploy (rotates all generated secrets)

task secrets:edit:prod  # open prod.sops.yaml in SOPS editor
task secrets:edit:local # open local.defaults.yaml in $EDITOR
```

## Structure

```
home-stack/
  Taskfile.yml
  docker/
    docker-compose.yml        # base stack, prod-oriented
    docker-compose.local.yml  # local overrides: HTTP only, relaxed auth
  traefik/
    traefik.yml               # static config: entrypoints, JWT plugin, ACME
    local/
      traefik.yml             # local static config: HTTP only, DEBUG log
    dynamic/
      middlewares.yml         # JWT, ForwardAuth, oauth2-proxy, strip-prefix
  ansible/
    bootstrap.yml             # ONE-TIME manual bootstrap (see file header)
    site.yml                  # CI deploy: clone repo + install env + start stack
    group_vars/all.yml        # non-secret defaults
    inventory/
      prod.yml                # reads VPS_HOST from env
      local.yml               # localhost connection
    roles/
      docker/                 # installs Docker Engine + Compose plugin
      common/                 # host hardening: UFW, fail2ban, unattended-upgrades
      stack/                  # clones repo, installs runtime env file
      systemd/                # installs home-stack.service unit
  terraform/
    main.tf                   # providers + backend
    variables.tf
    secrets.tf                # secret merging, random_password resources
    keycloak.tf               # realm, clients, DCR, users
    postgres.tf               # roles, databases
    dotenv.tf                 # assembles /etc/home-stack.env
    outputs.tf
    envs/
      local.tfvars
      prod.tfvars
    scripts/
      upsert_keycloak_dcr.py  # DCR policy upsert (subType + trusted-hosts)
      local_smoke_test.py     # end-to-end local validation
  secrets/
    local.defaults.yaml       # committed plaintext defaults
    prod.sops.yaml            # SOPS-encrypted production secrets
  .github/
    workflows/
      ci.yml                  # format + lint + validate on PRs to main
      provision.yml           # decrypt secrets + tofu apply + ansible deploy on push to main
      bootstrap.yml           # one-time VPS bootstrap (workflow_dispatch only)
```

## Adding a new service

1. Add a `services:` block in `docker/docker-compose.yml`
2. Attach to `public` network (and `internal` if it calls other services)
3. Add Traefik labels following the existing pattern
4. Add any generated secrets as `random_password` resources in `terraform/secrets.tf`
5. Add the runtime env vars to `terraform/dotenv.tf`
6. Add the service vars to `secrets/local.defaults.yaml`
7. Add a `watchtower.enable=true` label if it's a first-party image

## First-time production setup

### Configure (all must be done before bootstrap)

**`secrets/prod.sops.yaml`** — `task secrets:edit:prod`, then commit
- [ ] `domain` — bare domain, e.g. `example.com`
- [ ] `acme_email` — Let's Encrypt notification address
- [ ] `pg_admin_password` — stable tofu entry-point credential. Set it once. To rotate later: (1) `docker exec -it home-stack-postgres-1 psql -U home_stack_admin -c "ALTER ROLE home_stack_admin PASSWORD 'newpassword'"` on the VPS, (2) update this value, (3) `task rotate`.
- [ ] `keycloak_admin_password` — stable tofu entry-point credential. Set it once. To rotate later: (1) change it via the Keycloak admin UI at `auth.<domain>/admin`, (2) update this value, (3) `task rotate`.
- [ ] `keycloak_admin_allowed_ips` — CIDR allowlist for `/admin`, e.g. `203.0.113.10/32`
- [ ] `google_identity_provider.enabled` / `client_id` / `client_secret` / `hosted_domain` — set `enabled: false` to skip

**`ansible/group_vars/all.yml`** — edit and commit
- [ ] `ci_ssh_public_key` — public half of `VPS_SSH_DEPLOY_KEY`; bootstrap installs it into `deploy@vps:~/.ssh/authorized_keys`

**GitHub repo → Settings → Secrets**
- [ ] `AGE_PRIVATE_KEY` — full text of the age private key (`age-keygen -o ~/.config/age/home-stack.key`, then `cat` it)
- [ ] `VPS_SSH_DEPLOY_KEY` — SSH private key CI uses to reach the `deploy` user (Ansible + Postgres tunnel)
- [ ] `VPS_HOST` — VPS IP address
- [ ] `B2_ACCESS_KEY_ID` — Backblaze application key ID (create a private bucket + app key via B2 console)
- [ ] `B2_SECRET_ACCESS_KEY` — Backblaze application key secret

**GitHub repo → Settings → Variables**
- [ ] `TOFU_STATE_BUCKET` — B2 bucket name
- [ ] `TOFU_STATE_ENDPOINT` — e.g. `https://s3.us-west-004.backblazeb2.com`
- [ ] `TOFU_STATE_REGION` — e.g. `us-west-004`
- [ ] `TOFU_STATE_KEY_PREFIX` — e.g. `homestack`

**VPS** — via hosting panel
- [ ] `debian` user with passwordless sudo
- [ ] Your SSH public key in `debian@vps:~/.ssh/authorized_keys`

### Run

- [ ] `task bootstrap` — one-time, connects as `debian`, creates `deploy` user, installs Docker, hardens host, clones repo
- [ ] Push to `main` — `provision.yml` runs: decrypt → `tofu apply` → `ansible site.yml` → stack is live

### Local operator overrides (for manual prod ops)

- [ ] `SOPS_AGE_KEY_FILE=~/.config/age/home-stack.key` in your shell — needed for `task provision` and `task secrets:edit:prod`
- [ ] SSH key matching `VPS_SSH_DEPLOY_KEY` available locally — needed for `task bootstrap` and `task deploy`

---

**Runtime env** (`/etc/home-stack.env` on VPS, `.generated/local/home-stack.env` locally) — generated by `tofu apply`, never edit manually:
- Generated by `random_password` (rotated by `task rotate`): `PG_KEYCLOAK_PASSWORD`, `OAUTH2_PROXY_CLIENT_SECRET`, `OAUTH2_PROXY_COOKIE_SECRET`
- Derived from `domain`: `KEYCLOAK_HOSTNAME`, `KEYCLOAK_ISSUER_URL`, `OAUTH2_PROXY_COOKIE_SECURE`, `OAUTH2_PROXY_REDIRECT_URL`
- Everything else passed through from SOPS secrets

## CI/CD

Push to `main` triggers `provision.yml`: decrypt SOPS secrets → tofu apply → ansible site.yml (provision + deploy in one workflow). PRs to `main` run `ci.yml` (format + lint + validate only). `bootstrap.yml` is `workflow_dispatch` only — run it once per new VPS before any CI runs.
